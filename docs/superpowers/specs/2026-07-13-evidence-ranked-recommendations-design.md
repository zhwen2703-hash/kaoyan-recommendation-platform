# 考研院校证据分级推荐设计

## 目标

帮助用户在先填报志愿、后参加初试的约束下，从全部统考 408 学校专业中快速得到可解释的冲、稳、保候选，同时保留完整结果。推荐指数是相对排序工具，不是录取概率。

## 核心原则

1. 数据真实性优先于榜单完整性。
2. 专业线优先级为：专业线、学院共用线、学校线、国家线。
3. 只有国家线或关键数据缺失时，结果标为“待核验”，不能标为稳或保。
4. 缺失指标不按中等值补齐；按现有有效指标重新分配权重，并施加不确定性扣分。
5. 所有推荐理由和风险提示必须能追溯到页面中的数据与来源。
6. 推荐结果不得保存为静态榜单；每次查询必须基于当前最新数据快照实时重算。

## 动态更新

- GitHub Actions 每 6 小时检查并同步招生单位与严格 408 专业数据。
- 支持在 GitHub Actions 中手动触发立即同步。
- 数据快照变更后自动重新部署公网网站，原子切换到完整新快照。
- API 使用 `no-store`，每次筛选和推荐查询都从当前部署的数据版本重新计算。
- 招生人数、复试线、复录比、扩招状态或来源置信度变化时，推荐指数、排序、冲稳保和推荐理由必须随之变化。
- 页面显示数据版本时间；超过 12 小时未更新时显示“数据可能陈旧”，但继续提供上一版完整结果。
- 同步失败时不发布半成品数据，继续使用上一版已验证快照并显示同步状态。

## 用户流程

1. 用户输入预期总分。
2. 用户选择地区、英语科目、数学科目、院校层级等常用条件。
3. 系统展示 Top 10 推荐，按冲、稳、保或待核验分组。
4. Top 10 下方继续展示全部符合条件的学校专业，支持完整高级筛选、分页和排序。
5. 用户打开来源核验专业线、招生人数和复录比后再确定志愿。

## 筛选界面

基础筛选常驻显示：

- 预期总分
- 地区
- 英语一/英语二
- 数学一/数学二
- 985/211/双一流/普通
- 查询按钮

高级筛选折叠显示：

- 复录比上限
- 复试线参考上限
- 本科背景网络口碑
- 最低拟招生人数
- 扩招、持平、缩招、2027 未公布
- 排序方式

## 数据证据等级

- A 级：学校或学院官网专业线、官方复试名单与拟录取名单、明确专业招生计划。
- B 级：学院共用线、学校复试基本线、两条以上一致的可信来源。
- C 级：单一可信研招机构整理的网络参考。
- D 级：国家线兜底或关键字段缺失。

页面将证据等级转换为高、中、低置信度，并显示具体来源类型。

## 推荐算法

### 第一步：资格过滤

严格应用考试科目、地区、院校层级、学位类型、最低招生人数等硬条件。统考 408 必须为第四科代码精确等于 408。

### 第二步：指标计算

- 分数匹配：预期分数减去最高可信复试门槛。
- 复试友好度：根据复录比计算，越接近 1:1 越有利。
- 招生规模：使用最新已公开专业计划，按学校、学院、专业、培养方式去重。
- 招生趋势：只有 2026 与 2027 均公开时才计算扩招、持平或缩招。
- 院校竞争度：985、211、双一流作为弱因子，不覆盖实际专业数据。
- 数据完整度：专业线、复录比、招生人数、年度趋势的有效数据比例。

### 第三步：动态权重

基础权重：

- 分数匹配 40%
- 复录比 25%
- 招生规模 20%
- 招生趋势 10%
- 院校竞争度 5%

某项缺失时不为该项赋中性分，而是将其权重按比例分配给其他有可靠证据的指标，并根据证据等级扣除 5 至 25 分不确定性分。

### 第四步：风险分档

- 保：可靠专业线分差不少于 30 分、复录比不高于 1.3、专业计划不少于 10 人，且无明显缩招。
- 稳：可靠专业线分差为 10 至 29 分，招生规模和复录比没有明显风险。
- 冲：分差不足 10 分、低于参考线、招生计划较少或竞争因素明显。
- 待核验：仅有学校线、国家线，或专业线/复录比等关键证据不足。

“待核验”不参与保底推荐。

### 第五步：Top 10 多样性

- 同一学校最多展示 2 个专业。
- 优先形成 3 个冲、4 个稳、3 个保的组合；某档证据不足时不强行凑数。
- 待核验候选单独展示，不占用可信 Top 10 名额。

## 推荐结果

每条 Top 10 结果显示：

- 学校、学院、专业、学硕/专硕
- 推荐指数和冲/稳/保/待核验
- 数据置信度
- 预期分与参考线分差
- 复录比、招生人数、扩招趋势
- 2 至 4 条推荐理由
- 1 至 3 条风险提示
- 数据来源入口

## 错误与边界

- 预期分数必须在 200 至 500 之间，否则不计算个人分数匹配。
- 2027 招生人数未公布时，不计算扩招。
- 网络口碑和网络复录比必须标注非官方属性及置信度。
- API 失败时保留上一次完整结果并显示更新时间，不展示半份榜单。
- 国家线只作为最低门槛，不能称为学校专业线。

## 验证

- 单元测试覆盖动态权重、数据缺失、风险分档和同校去重。
- 验证只有国家线的结果永远不会进入稳或保。
- 验证缩招、低招生人数和高复录比会降低排名。
- 验证提高预期分数不会降低同一专业的分数匹配得分。
- 验证 Top 10 同校不超过 2 条。
- 修改测试快照中的复录比、招生人数、复试线和扩招数据，验证推荐分与排序发生符合预期的变化。
- 验证相同筛选条件在数据版本变化后不会返回旧缓存榜单。
- 运行 lint、测试、生产构建和公网 API 检查。
- 桌面端和移动端检查基础筛选、高级筛选、Top 10、全部结果和横向滚动行为。
